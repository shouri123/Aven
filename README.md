## 🌟 Project Overview

Traditional mental wellness applications fail to balance active engagement with strict clinical safety. Chatbots generated by traditional Large Language Models (LLMs) often suffer from diagnostic hallucinations and lack persistent stateful memory. 

**Aven** solves this by completely abstracting generative intelligence into a multi-node, hierarchical pipeline. It provides a quiet, minimalist, chat-first interface where the AI dynamically unlocks highly sophisticated reasoning matrices, pattern tracking, and data visualizations entirely based on user interaction volume.

---

## 🏗️ Hierarchical Multi-Agent Mental Wellness Architecture (MAMWA)

Aven does not rely on a single conversational prompt. Instead, inputs are pipelined across a deeply specialized network of independent Agentic Nodes. This isolation guarantees that logical deduction is mathematically separated from text generation, virtually eliminating clinical hallucinations.

### 🔄 The Execution Pipeline Architecture
```text
[User Client] ── Input ──> [Orchestrator Route]
                                  │
                                  ├─> 🚨 Risk Agent (Immediate Halt on Detection)
                                  ├─> 💾 Memory Agent (Local Vector Context)
                                  ├─> 📊 Pattern Agent (Streak & Burnout Calc)
                                  ├─> 🧠 Emotion Agent (Signal & Sentiment Parsing)
                                  ├─> ⚙️ Reasoning Agent (Mathematical Logic & Strategy)
                                  └─> 💬 Response Agent (Empathic Prose Generation)
                                  │
[User Client] <── JSON Array <────┘
```

---

## ⚙️ Agent Capabilities & Lifecycle

Every interaction cascades through six micro-agents designed for extreme accuracy:

1. **🚨 Risk Agent**: Operates outside generative logic. Uses deterministic heuristic parsing to intercept signs of severe distress, abuse, or self-harm. If triggered, it short-circuits the entire pipeline instantly to return certified emergency protocols.
2. **🧠 Emotion Agent**: A constrained extraction layer. Evaluates raw text to identify primary/secondary emotions, returning strict JSON vectors rather than prose.
3. **💾 Memory Agent**: Interfaces with the user's secure browser database to inject historical context. Allows Aven to recall names, past distress topics, and previous conversational threads securely.
4. **📊 Pattern Agent**: Calculates complex psychological behaviors over time (e.g., detecting if a user has logged specific negative sentiments for 3+ consecutive days).
5. **⚙️ Reasoning Agent**: The logical brain. Takes all previous context and synthesizes an objective, clinically-safe deduction of the user's state. Creates non-diagnostic strategies.
6. **💬 Response Agent**: The only agent tasked with empathy. Consumes the Reasoning Agent's factual conclusions and wraps them in a highly organic, warm, and highly personalized textual response.

## 🚀 Core Features & UX Patterns

Aven features a heavily researched psychological UX designed for engagement and self-reflection:

- **AI Reasoning Transparency**: Users can explicitly click into a "Reasoning Panel" in the UI to view the exact algorithmic chain the agents used to generate their response.
- **Proactive Intelligence**: Aven detects negative momentum over time, generating proactive "Check-In" signals unprompted when identifying long-term exhaustion or anxiety.
- **Progressive UI Disclosure**:
  - **Zero to 2 Entries**: The interface remains quiet, acting strictly as a conversational listener.
  - **3 Entries**: Unlocks **AI Emotional Insights**, highlighting dominant detected themes.
  - **5 Entries**: Unlocks the **Interactive Wellness Dashboard** (SVG pattern mapping).
  - **7 Entries**: Unlocks **Advanced Behavioral & Medical Guidance**.

## 🔒 Security, Privacy & Clinical Guardrails

Aven is strictly designated as a proactive wellness companion, built upon rigid medical software guardrails:

- **Zero Remote Storage**: User journal vectors, authentication hashes, and psychological insights are strictly processed and stored in client-side HTML5 `localStorage`. No centralized persistent cloud database holds raw user text.
- **Diagnostic Prohibition**: The internal Engine Prompts enforce absolute bans on diagnostic formatting or prescription suggestions.
- **Offline & API Fallbacks**: The Orchestrator wraps all asynchronous API communication inside robust Error Boundaries. If upstream rate limits are intercepted, the system smoothly defaults to localized safe prompts without breaking the user experience.

## 🛠️ Local Deployment & Engineering Setup

### Prerequisites
- Node.js (v18+)
- Active OpenAI Developer Account & API Key

### Installation

1. **Clone the repository**:
   ```bash
   git clone https://github.com/shouri123/AURA.git aven-wellness
   cd aven-wellness
   ```

2. **Install core dependencies**:
   ```bash
   npm install
   ```

3. **Configure Environment Variables**:
   Create a `.env.local` file at the root of the project to initialize the Orchestrator with OpenAI credentials:
   ```env
   OPENAI_API_KEY=sk-your_production_key_here
   ```

4. **Initialize Local Development Server**:
   ```bash
   npm run dev
   ```

5. **Deploy**:
   The application is fully compatible with Vercel or standard Next.js optimized build servers. Navigate to [http://localhost:3000](http://localhost:3000) to begin the interactive session.
